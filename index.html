02 — Intelligence

Baseline Comparison Logic

Unlike naive scrapers that dump raw totals, this bot calculates the delta — giving you subject-by-subject accountability for the day.

01
Morning Calibration — 09:00 AM
The bot scans your portal, extracts all enrolled subjects, and saves the current attended/held counts per subject as the day's baseline in last_attendance.txt.
02
Delta Calculation
Every subsequent run computes Current − Baseline for each subject. This isolates today's classes from your semester history.
03
Itemized Telegram Report
Only subjects with new activity are listed. Your true combined percentage is computed from total semester data and appended beneath.

03 — Configuration

Setup Guide

Navigate to Settings → Secrets and variables → Actions in your GitHub repo to add all credentials.
01 Fork Repository 02 Create Telegram Bot 03 GitHub Secrets 04 Initialize JSON
1
Fork the Repository

Click Fork at the top-right of the GitHub repository page. This creates your personal copy where all secrets and workflows will live.

2
Create Your Telegram Bot

You need a Telegram Bot Token (BOT_TOKEN) and your personal Chat ID (CHAT_ID) before filling in GitHub Secrets. Follow these steps carefully.

Part A — Get Your BOT_TOKEN
1
Open Telegram and search for BotFather
Search @BotFather in the Telegram search bar. It's the official verified bot with a blue checkmark ✓. Tap on it and press Start.
2
Send the /newbot command
Type and send /newbot. BotFather will ask you to choose a display name for your bot.
3
Choose a username
It must be unique and end with bot (e.g. my_attendance_bot).
4
Copy your Bot Token
BotFather will reply with a message containing your token. Copy this entire string. This is your BOT_TOKEN. Never share it publicly.
5
Start a chat with your new bot
Search your bot's username in Telegram and press Start. The bot cannot send you messages until you initiate a conversation.
Part B — Get Your CHAT_ID
1
Search for @userinfobot
In Telegram, search @userinfobot and press Start.
2
Copy your ID
The bot will instantly reply with your details. Copy the number next to Id: — that is your CHAT_ID.
3
Configure GitHub Secrets

All sensitive credentials are encrypted inside GitHub Secrets — never exposed in logs or code. Add each of the following in your repo's Settings → Secrets and variables → Actions.

Secret NameValue
USER_IDYour portal username / roll number
PASSWORDYour portal password
BASE_URLPortal URL (e.g. https://arms.sse.saveetha.com). Defaults to SMC if blank.
BOT_TOKENToken copied from @BotFather
CHAT_IDYour ID copied from @userinfobot
GH_PATGitHub Personal Access Token (Classic) with workflow scope
REPO_FULL_NAMEYour repository path — e.g. username/attendancereporter
4
Initialize JSON Memory

Edit last_attendance.txt, clear all contents, and set it to exactly this empty JSON object:

{}

On its first run, the bot auto-discovers all your subjects from the portal and builds a full memory map. The empty JSON is simply the starting point.

04 — Optional Upgrade

Refresh Now
Button Setup

GitHub Actions can't natively listen to Telegram callbacks. A Cloudflare Worker acts as the bridge — receiving the button tap and triggering the GitHub workflow via API.

Skipping this phase?

The bot will still run automatically 3 times daily. Just avoid tapping "Refresh Now" in Telegram — without the listener, the request won't be processed.
01
Create & Deploy Worker
Log in to Cloudflare → Workers & Pages → Create application → Create Worker → Deploy → Edit Code.
02
Paste Webhook Script
Replace all content in worker.js with the provided webhook script from the repository.
03
Add Environment Variables
Under Settings → Variables, add: BOT_TOKEN, GH_PAT, and REPO_FULL_NAME.
04
Activate Webhook
Copy your Worker URL and open this in a browser to connect Telegram to the listener:

https://api.telegram.org/bot<TOKEN>/
setWebhook?url=<WORKER_URL>

Common Questions

Is my password safe?
Yes. Credentials are encrypted inside GitHub Secrets and never appear in any logs or console output.
What happens on a holiday?
The bot detects zero new classes for all subjects and updates the message to say "No classes updated yet today."
Can I trigger it manually?
Yes. Go to the Actions tab in your GitHub repository, select the workflow, and click Run workflow.
